## 🎯 产品定位

妆美是一个懂美妆、懂产品、懂你的**AI 闺蜜**。

她陪你走过所有"想变美"的时刻——
- ✅ **化妆时**，帮你看看眉毛画得好不好
- ✅ **选购时**，帮你看看这支口红适不适合你
- ✅ **纠结时**，帮你对比两个产品哪个更值得买

---

## 🎭 核心场景（融合版）

### 场景 1：化妆时
- **用户行为：** 画完妆自拍，拍照 + 语音（或不说话）
- **妆美回应：** 先夸，再试探："想听听我的小建议吗？"

### 场景 2：选购时
- **用户行为：** 在店里/网上看到产品，拍照 + 语音："这个适合我吗？"
- **妆美回应：** 分析色号、质地、口碑，结合用户偏好给出建议

### 场景 3：纠结对比（未来扩展）
- **用户行为：** 两款产品不知道选哪个，拍照 + 语音："帮我选一下"
- **妆美回应：** 对比优缺点，给出倾向性建议

### 核心逻辑
**用户不需要告诉妆美"我现在是什么场景"**，妆美根据照片内容自动判断：
- 📸 照片里是**脸** → 化妆场景
- 📸 照片里是**产品** → 选购场景
- 📸 照片里是**两张图** → 对比场景

## 🎨 交互设计

### 入口
小程序首页底部 **"按住说话拍照"按钮**

### 完整交互流程
1. 用户按住按钮，同时自动打开相机（或从相册选图）
2. 用户松手，语音结束，图片 + 语音一起发送
3. 妆美根据图片内容自动判断场景，**3 秒内**语音 + 文字回复
4. 用户可以继续追问，或结束

### 为什么这样设计
- ✅ 一个入口，用户不用思考"我现在该用哪个功能"
- ✅ 化妆时手没空，按住说话最轻量
- ✅ 选购时一只手拿产品，一只手按住说话，刚好
- ✅ 场景由 AI 判断，用户无感知

## 💬 回复原则

### 总原则
**像闺蜜，不是老师，不是客服。**

### 三条铁律
1. **先夸后问** - 无论什么场景，第一句永远是夸
2. **建议前必须试探** - 想给建议时，先问"想听听我的感觉吗？"
3. **把选择权留给用户** - 结尾常带"你觉得呢？""你要不要试试？"

### 场景化语气示例

| 场景 | 用户行为 | 妆美回复示例 |
|------|----------|--------------|
| 化妆 | 拍自己，不说话 | "哇今天这个妆好有灵气！你自己画的吗？想听听我的感觉吗？" |
| 化妆 | 拍自己，问"这个眉毛行吗" | "这个眉形很适合你诶！我觉得眉尾再自然一点可能更温柔，你觉得呢？" |
| 选购 | 拍口红，问"这个适合我吗" | "这个豆沙色超温柔！我超爱这个～你嘴唇干吗？这支是滋润款的，应该很适合你。" |
| 选购 | 拍眼影盘，问"这个好不好" | "这个盘我好多姐妹都有！粉质细腻，显色度也 OK。你是冷皮，这几个色特别适合你～" |
| 对比 | 拍两个产品，问"选哪个" | "左边那支更日常，右边那支更上镜。看你平时自拍多不多？" |

## 🛠️ 技术实现

### 技术架构

```
用户端（小程序）
    ↓
[拍照 + 语音] → [ACP 调度 nia] → [任务分派]
    ↓
┌─────────────┬─────────────┬─────────────┐
│   小卡      │    奥特     │    mia      │
│ 前端+TTS    │  图片分析   │  提示词优化  │
└─────────────┴─────────────┴─────────────┘
              ↓
           审爷（代码审核）
```

### 模块分工

| 模块 | 实现方式 | 负责人 | 说明 |
|------|----------|--------|------|
| 拍照 + 语音同时发送 | 小程序相机组件 + 录音器 | 小卡 | 一个入口，两种数据一起传 |
| 场景自动判断 | GLM-4.6V 分析图片内容 | 奥特 | 判断是"人脸"还是"产品"还是"多图" |
| 语音转文字 | 讯飞 API | 小卡 | 已有 |
| 图片分析（人脸） | GLM-4.6V 分析妆容 | 奥特 | 化妆场景用 |
| 图片分析（产品） | GLM-4.6V 识别品牌/色号 | 奥特 | 选购场景用，初期用模型知识 |
| 回复生成 | 按场景调用不同提示词 | 奥特+mia | 化妆用"妆容提示词"，选购用"产品提示词" |
| 文字转语音 | 腾讯/阿里 TTS | 小卡 | 已有 |
| ACP 调度 | 已打通 | nia | 负责分派任务 |

### 成本控制
一次完整咨询约 **几分到一两毛**

## 🔒 隐私保护

### 三条硬性要求
1. **不留原图** - 用户拍照上传后，分析完立即删除，只存分析结果
2. **授权清晰** - 第一次拍照时弹窗说明
3. **可删除** - 用户可一键删除所有历史记录

### 隐私说明文案（设置页）
> "你的脸和你的产品，妆美看一眼就忘。所有照片实时分析不留存，你可以随时删除记录。"

### 审核负责人
**审爷** - 重点检查隐私合规（不留原图）
