# ACP 故障预防制度

**创建时间：** 2026-03-19  
**触发事件：** acpx 版本不匹配导致 ACP 不可用 3 小时

---

## 📋 故障回顾

**时间：** 2026-03-19  
**影响：** ACP 不可用，无法用 `sessions_spawn` 调用子 agent  
**根因：** acpx 版本不匹配（期望 0.1.16，实际 0.3.0）

---

## 🔧 预防措施

### 1. 版本检查（每周）
```bash
# 检查 acpx 版本
acpx --version

# 检查 openclaw 期望版本
# 位置：openclaw/extensions/acpx/openclaw.plugin.json
```

### 2. 配置备份
- `.env.local` → 加密存储
- `openclaw.json` → GitHub 私有仓库
- `~/.acpx/config.json` → 备份到共享记忆区

### 3. 应急预案
**ACP 不可用时的临时方案：**
- ✅ 飞书消息联系 Agent
- ✅ 共享记忆区读写任务
- ✅ webchat 直接对话

---

## 📝 故障报告流程

1. **发现问题** → 立即上报主任
2. **临时方案** → 启用飞书 + 共享记忆区
3. **分析根因** → 奥特负责技术分析
4. **修复验证** → 测试通过后恢复 ACP
5. **写入记忆** → MEMORY.md + 故障报告

---

## ⚠️ 禁止行为

- ❌ 不检查版本就升级 openclaw
- ❌ 不测试配置就重启网关
- ❌ 故障时不启用临时方案
- ❌ 不写故障分析报告

---

**制度来源：** 主任指示"避免下次再犯"
