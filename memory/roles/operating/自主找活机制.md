**生效日期：** 2026-03-19  
**状态：** ✅ 已激活

## 🎯 核心原则（马斯克管理模式）

1. **主任只提需求** - 不安排具体执行步骤
2. **全员自主找活** - 自己发现任务、自己执行、自己汇报
3. **结果导向** - 不看过程，看产出
4. **真实汇报** - 不假装努力，不胡编进度

## 🔍 任务发现流程

### 步骤 1：扫描任务区
```
1. 读取 /memory/roles/operating/tasks/ 下所有任务文件
2. 筛选状态为"待开始"或"进行中"的任务
3. 根据自己的角色匹配合适任务
```

### 步骤 2：评估优先级
```
P0 - 紧急重要（立即执行）
P1 - 重要不紧急（今天完成）
P2 - 日常优化（本周完成）
```

### 步骤 3：领取任务
```
1. 更新任务文件状态为"进行中"
2. 添加执行者名字
3. 添加开始时间
```

### 步骤 4：执行任务
```
1. 按任务要求执行
2. 遇到阻塞立即上报
3. 完成后更新状态为"已完成"
```

### 步骤 5：汇报结果
```
格式：
【任务完成汇报】
- 任务：[任务名]
- 执行者：[名字]
- 状态：✅ 已完成
- 产出：[文件/链接]
- 问题：[如有]
- 下一步：[建议]
```

## ⏰ 心跳检查

**每个 Agent 每次心跳时检查：**
- [ ] 任务区有无新任务
- [ ] 自己负责的任务有无阻塞
- [ ] 是否需要协调资源
- [ ] 有无高优先级市场机会

**心跳响应：**
- 无异常 → `HEARTBEAT_OK`
- 有异常 → 立即汇报

---

## 🚨 阻塞上报

**遇到以下情况立即上报：**
1. 技术难点超过 30 分钟未解决
2. 需要其他 Agent 协作
3. 需要主任决策
4. 资源/权限不足

**上报格式：**
```
🚨 阻塞报告
- 任务：[任务名]
- 执行者：[名字]
- 阻塞原因：[详细描述]
- 需要：[需要什么帮助/决策]
- 影响：[对进度的影响]
```
