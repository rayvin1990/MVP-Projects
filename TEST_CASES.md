# ContextCapsule 测试用例

## 一、核心流程测试

### 1.1 Save Summary 正常流程

| 用例 ID | CC-SAVE-001 |
|--------|-------------|
| **测试目标** | 验证 Save Summary 功能在正常场景下正确保存对话摘要 |
| **前置条件** | 1. 已安装 ContextCapsule 扩展<br>2. 浏览器已打开支持的对话页面（如 claude.ai）<br>3. 页面中存在对话历史 |
| **测试步骤** | 1. 点击扩展图标打开胶囊面板<br>2. 点击 "Save Summary" 按钮<br>3. 等待处理完成<br>4. 检查胶囊列表是否新增一条记录<br>5. 点击新增记录查看摘要内容 |
| **预期结果** | 1. 成功生成对话摘要<br>2. 胶囊列表中显示新记录（包含标题、时间戳）<br>3. 摘要内容准确反映对话要点<br>4. 显示成功提示消息 |
| **优先级** | P0 |

| 用例 ID | CC-SAVE-002 |
|--------|-------------|
| **测试目标** | 验证 Save Summary 后胶囊可正常查看和编辑 |
| **前置条件** | 已执行 CC-SAVE-001 并成功保存 |
| **测试步骤** | 1. 在胶囊列表中点击刚保存的胶囊<br>2. 查看摘要详情<br>3. 尝试编辑标题或内容<br>4. 保存修改 |
| **预期结果** | 1. 胶囊详情正确显示<br>2. 编辑功能可用<br>3. 修改后成功保存 |
| **优先级** | P1 |

## 四、附录

### 4.1 测试环境要求

- **浏览器:** Chrome 120+ 或同等 Chromium 内核
- **扩展版本:** ContextCapsule v1.0+
- **网络:** 稳定互联网连接（部分测试需断网模拟）

### 4.2 缺陷严重程度定义

| 等级 | 描述 |
|------|------|
| P0 | 阻塞性缺陷，核心功能不可用 |
| P1 | 严重缺陷，主要功能受影响但有 workaround |
| P2 | 一般缺陷，次要功能问题或边界场景 |
| P3 | 轻微缺陷，UI/UX 优化建议 |

### 4.3 测试通过标准

- 所有 P0 用例 100% 通过
- 所有 P1 用例 95%+ 通过
- P2 用例 80%+ 通过
- 无阻塞性缺陷遗留
