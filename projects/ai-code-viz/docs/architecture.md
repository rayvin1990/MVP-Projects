# AI Code Viz - 架构设计文档

## 1. 系统概述

AI Code Viz 是一个用于可视化代码库结构和依赖关系的工具，解决 Vibe Coding 后 AI 失去代码库心理模型的问题。

### 核心目标
- 可视化文件/函数/类之间的连接关系
- 支持多种编程语言
- 提供交互式探索界面
- 支持增量更新和大型代码库

## 3. 核心模块设计

### 3.1 解析器模块 (Parser Module)

**职责：** 将源代码转换为抽象语法树 (AST)

**工作流程：**
1. 文件发现：递归扫描目标目录，识别支持的语言文件
2. 语言检测：根据文件扩展名确定解析器
3. AST 生成：使用 Tree-sitter 生成语法树
4. 错误处理：捕获解析错误，记录日志，继续处理其他文件

**输出：** AST 树结构，包含节点类型、位置、父子关系

### 3.2 符号提取模块 (Symbol Extractor)

**职责：** 从 AST 中提取有意义的代码符号

**提取内容：**
- 函数定义（名称、参数、返回值、位置）
- 类/接口定义（名称、成员、继承关系）
- 变量声明（作用域、类型）
- 导入/导出语句（模块依赖）

**输出：** 符号表（Symbol Table），包含符号元数据

### 3.3 依赖图构建模块 (Dependency Graph Builder)

**职责：** 构建代码元素之间的关系图

**关系类型：**
- **调用关系：** 函数 A 调用函数 B
- **导入关系：** 文件 A 导入文件 B
- **继承关系：** 类 A 继承类 B
- **引用关系：** 函数引用变量/常量

**图结构：**
```typescript
interface CodeGraph {
  nodes: CodeNode[];
  edges: CodeEdge[];
}

interface CodeNode {
  id: string;
  type: 'file' | 'function' | 'class' | 'variable';
  name: string;
  path?: string;
  position?: Position;
  metadata: Record<string, any>;
}

interface CodeEdge {
  source: string;
  target: string;
  type: 'calls' | 'imports' | 'extends' | 'references';
}
```

### 3.4 布局引擎 (Layout Engine)

**职责：** 计算节点在画布上的位置

**布局策略：**
- **层级布局：** 按调用栈分层
- **力导向布局：** 自动调整节点间距
- **分组布局：** 按文件/模块分组

### 3.5 渲染引擎 (Rendering Engine)

**职责：** 在画布上绘制图形

**渲染内容：**
- 节点（圆形、矩形、图标）
- 边（箭头、曲线、颜色编码）
- 标签（名称、类型提示）
- 交互反馈（悬停、选中、高亮）

## 5. 性能优化设计

### 5.1 增量解析
- 监听文件系统变化（chokidar）
- 仅重新解析变更的文件
- 增量更新依赖图

### 5.2 Web Worker
- 解析和图构建在 Worker 线程执行
- 避免阻塞 UI 渲染
- 使用 MessageChannel 通信

### 5.3 虚拟滚动
- 仅渲染可视区域内的节点
- 大规模代码库支持（10000+ 节点）

### 5.4 图简化
- 折叠子图（按文件/模块）
- 层次化显示（展开/收起）
- 兴趣点聚焦（高亮路径）

## 6. 扩展性设计

### 6.1 语言支持
通过 Tree-sitter 语法包轻松添加新语言：
```typescript
const languages = {
  typescript: require('tree-sitter-typescript'),
  python: require('tree-sitter-python'),
  rust: require('tree-sitter-rust'),
  // 轻松扩展...
};
```

### 6.2 可视化主题
支持自定义节点样式、颜色方案、布局配置

### 6.3 导出格式
支持导出为 JSON、PNG、SVG 等格式

## 7. 技术约束
