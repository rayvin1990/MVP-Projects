### 2. Agent 角色详解

#### 🧠 主 Agent (Coordinator)
**别名:** 项目经理、协调员

**职责:**
- 接收用户任务
- 分析任务复杂度
- 分解为子任务
- 分配给合适的子 Agent
- 监督执行进度
- 汇总结果并汇报

**能力:**
- 任务规划
- 资源调度
- 冲突解决
- 质量控制

**触发条件:** 用户直接对话

---

**职责:**
- 代码编写

**专长:**
- Python/JavaScript 开发

- API 集成

**触发条件:** 主 Agent 分配开发任务

---

**职责:**
- 编写测试用例

**专长:**
- 单元测试

**触发条件:** 主 Agent 分配测试任务

---

## 🔄 工作流程

### 标准任务流程

```
1. 用户提交任务
   │
   ▼
2. 主 Agent 接收并分析
   │
   ├── 简单任务 ──► 主 Agent 直接处理
   │
   └── 复杂任务
       │
       ▼
3. 任务分解
   │
   ├── 子任务 A (开发) ──► Agent A
   ├── 子任务 B (测试) ──► Agent B
   └── 子任务 C (文档) ──► Agent C
   │
   ▼
4. 并行/串行执行
   │
   ▼
5. 结果收集
   │
   ▼
6. 质量检查
   │
   ├── 不合格 ──► 返工
   │
   └── 合格
       │
       ▼
7. 汇总报告
   │
   ▼
8. 提交用户
```

---

### 自主规划流程

```
1. 主 Agent 接收目标
   │
   ▼
2. 目标分析
   │
   ├── 明确目标 ──► 直接规划
   │
   └── 模糊目标 ──► 询问用户澄清
       │
       ▼
3. 制定计划
   │
   ├── 阶段 1: 需求分析
   ├── 阶段 2: 设计方案
   ├── 阶段 3: 实现开发
   ├── 阶段 4: 测试验证
   └── 阶段 5: 文档交付
   │
   ▼
4. 资源分配
   │
   ├── Agent A: 阶段 2-3
   ├── Agent B: 阶段 4
   └── Agent C: 阶段 1,5
   │
   ▼
5. 执行监控
   │
   ├── 进度正常 ──► 继续
   │
   └── 进度延迟 ──► 调整资源/计划
       │
       ▼
6. 结果交付
```

## 📋 任务分配策略

### 1. 基于技能分配

| 任务类型 | 分配给 | 理由 |
|----------|--------|------|
| 代码编写 | Agent A | 开发专长 |
| 测试验证 | Agent B | 测试专长 |
| 文档编写 | Agent C | 写作专长 |
| 架构设计 | Agent A + 主 Agent | 需要综合视角 |
| Bug 修复 | Agent A + Agent B | 开发 + 验证 |

### 2. 基于负载分配

```
Agent A: 当前任务 2 个 (容量 5) - 可分配
Agent B: 当前任务 4 个 (容量 5) - 接近满载
Agent C: 当前任务 1 个 (容量 5) - 可分配

新任务 → 优先分配给 Agent A 或 C
```

### 3. 基于优先级分配

```
P0 (紧急) → 立即分配，可中断低优先级任务
P1 (高)   → 优先分配
P2 (中)   → 正常排队
P3 (低)   → 空闲时处理
```

## 🎯 自主决策机制

### 1. 任务自动分配规则

```python
def assign_task(task, agents):
    # 1. 识别任务类型
    task_type = classify_task(task)
    
    # 2. 筛选合适的 Agent
    candidates = [a for a in agents if task_type in a.skills]
    
    # 3. 选择负载最低的
    best_agent = min(candidates, key=lambda a: a.current_load)
    
    # 4. 分配任务
    return best_agent
```

### 2. 冲突解决策略

```
冲突场景：多个任务同时需要同一 Agent

解决策略:
1. 优先级高的优先
2. 优先级相同，截止日期早的优先
3. 都相同，先到的优先
4. 必要时向用户请求增派资源
```

### 3. 异常处理

```
Agent 离线 → 自动重新分配任务给其他 Agent
任务失败 → 分析原因，决定重试或转派
进度延迟 → 调整计划，通知用户
质量不达标 → 返工或换人
```
