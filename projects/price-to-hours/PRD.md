# 价格转工时计算器 - 产品需求文档

## 项目信息
- **项目名称：** Time for Price（价格转工时）
- **创建日期：** 2026-03-06
- **目标用户：** 消费者、购物决策者
- **核心价值：** 帮助用户判断商品"值不值"

---

## 一、用户故事

### 主用户故事
> 作为消费者，我希望看到商品价格时能立刻知道"这值得我工作多久"，这样我能做出更理性的购买决策。

### 具体场景
1. **浏览购物网站** → 看到商品价格 → 悬停鼠标 → 看到工时
2. **首次使用** → 安装扩展 → 设置时薪 → 开始使用
3. **调整时薪** → 收入变化 → 打开设置 → 更新数值

---

## 二、功能列表

### MVP（第一版）
- ✅ 鼠标悬停价格显示工时

- 🔄 购买历史记录

- 🔄 多时薪模式（工作日/周末不同）

## 四、技术约束

### 技术栈
- **Chrome Extension** (Manifest V3)
- **Vanilla JS**（无框架，保持轻量）
- **Chrome Storage API**（本地存储设置）

### 文件结构
```

├── popup.html         # 设置页
```

### 核心逻辑

#### 价格识别
```javascript
// 正则表达式匹配价格
const priceRegex = /¥?\s?(\d+\.?\d*)|(\$|£|€)\s?(\d+\.?\d*)/g;
```

#### 工时计算
```javascript
function calculateHours(price, hourlyRate) {
  return (price / hourlyRate).toFixed(2);
}
```

#### 存储结构
```javascript

  hourlyRate: 50,
  currency: 'CNY',
```

## 七、风险和缓解

### 风险 1: 价格识别不准确
**缓解：**
- 使用多种正则表达式模式
- 支持常见货币符号
- 添加"手动选择"功能

### 风险 2: 网站样式冲突
**缓解：**
- 使用 Shadow DOM 隔离样式
- 添加 CSS class 前缀
- 测试主流网站

### 风险 3: 性能问题
**缓解：**
- 使用防抖（debounce）处理悬停
- 延迟加载内容脚本
- 优化正则表达式
