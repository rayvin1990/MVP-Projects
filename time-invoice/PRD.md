# TimeInvoice - 产品需求文档

## 1. 项目概述

**项目名称：** TimeInvoice（发票工具 2.0）

**项目类型：** 纯前端 Web 应用（SaaS）

**核心功能：** 计时 + 发票一体化，帮助按小时收费的自由职业者自动计算工时金额并生成专业发票

**目标用户：** 按小时收费的自由职业者（咨询师、开发者、设计师、律师等）

**版本：** 2.0（基于 FastInvoice 1.0 升级）

---

## 2. 产品定位

### 2.1 核心价值

| 1.0 传统方式 | 2.0 TimeInvoice |
|-------------|-----------------|
| 手动填写金额 | 计时 → 自动算金额 |
| 事后补录工时 | 实时记录 |
| 容易忘记计费 | 计时器常驻 |

### 2.2 用户痛点

| 痛点 | 频率 | 解决方案 |
|------|------|----------|
| 手动计算工时和金额 | 每次 | 计时器自动计算 |
| 忘记记录工时 | 经常 | 随时启动计时 |
| 多个项目时间混淆 | 经常 | 按项目分类记录 |
| 费率不同难以区分 | 有时 | 支持多费率设置 |

## 5. 数据模型

### 5.1 TimeEntry（时间记录）

```typescript
interface TimeEntry {
  id: string;              // UUID
  projectId: string;       // 关联项目
  description: string;     // 工作描述
  startTime: string;       // 开始时间 ISO
  endTime: string;         // 结束时间 ISO
  duration: number;        // 时长（秒）
  rate: number;            // 时薪
  amount: number;          // 金额 = duration / 3600 * rate
  invoiced: boolean;       // 是否已开票
  invoiceId?: string;      // 关联发票ID
  createdAt: string;
}
```

### 5.2 Project（项目）

```typescript
interface Project {
  id: string;
  name: string;            // 项目名称
  clientId?: string;       // 关联客户
  rate: number;            // 时薪（可覆盖默认）
  color: string;           // 项目颜色标签
  active: boolean;         // 是否活跃
  createdAt: string;
}
```

### 5.3 Invoice（发票）- 复用

```typescript
// 复用 FastInvoice 1.0 数据模型
interface Invoice {
  id: string;
  invoiceNumber: string;
  clientId: string;
  items: InvoiceItem[];
  timeEntryIds: string[];  // 关联的时间记录
  // ... 其他字段同 1.0
}
```

### 5.4 Settings（设置）

```typescript
interface Settings {
  defaultRate: number;     // 默认时薪
  currency: string;       // 货币
  userName: string;       // 用户名称
  userEmail: string;      // 用户邮箱
  userAddress: string;    // 用户地址
}
```

## 6. 核心交互

### 6.1 计时器交互

| 操作 | 触发 | 结果 |
|------|------|------|
| 开始 | 点击开始按钮 | 计时开始，显示实时时长 |
| 暂停 | 点击暂停按钮 | 计时暂停，可恢复 |
| 停止 | 点击停止按钮 | 结束计时，弹出保存表单 |
| 选择项目 | 下拉选择 | 切换当前计时的项目 |

### 6.2 时间记录 → 发票

| 操作 | 说明 |
|------|------|
| 多选 | checkbox 选择多条时间记录 |
| 合并 | 合并为一张发票 |
| 自动计算 | 金额 = Σ(时长 × 费率) |
| 跳转 | 创建后跳转到发票详情 |

## 8. 技术实现

### 8.1 技术栈

| 技术 | 选择 | 原因 |
|------|------|------|
| 框架 | React + Vite | 快速开发 |
| 样式 | TailwindCSS | 轻量 |
| 路由 | react-router-dom | 标准 |
| 存储 | localStorage | 无后端 |
| PDF | jsPDF + jspdf-autotable | 免费 |
| 邮件 | EmailJS | 免费额度 |
| 部署 | Vercel | 免费 |

### 8.2 复用策略

- **直接复用：** 发票创建逻辑、PDF 导出、客户管理
- **适配对接：** 时间记录 → 发票明细转换
- **新增开发：** 计时器、时间记录管理

### 8.3 数据流

```
用户操作 → React State → localStorage 持久化
                ↓
         计时器 Web Worker（后台计时）
                ↓
         时间记录列表
                ↓
         选择条目 → 生成发票
                ↓
         PDF 导出 / 邮件发送
```
