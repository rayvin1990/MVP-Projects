## 一、执行摘要

### 核心问题
多 Agent 协作系统中，长对话历史导致：
- Token 消耗激增（成本问题）
- 上下文窗口溢出（技术问题）
- 关键信息稀释（效果问题）

### 推荐方案
**混合压缩策略** = 摘要压缩 + 向量检索 + 关键信息保留

**技术栈选型：**
- 核心框架：LangChain Memory + 自定义压缩器
- 向量存储：ChromaDB（轻量级，嵌入式）
- 摘要模型：Qwen-Max（已有 API 集成）
- 嵌入模型：text-embedding-3-small（高性价比）

**预计开发工时：** 3-5 天（MVP）

## 二、技术调研

### 2.1 上下文压缩技术分类

| 技术 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| **摘要压缩** | LLM 生成对话摘要 | 保留语义，压缩率高 | 可能丢失细节，需要额外 API 调用 | 长对话历史归档 |
| **向量检索** | 嵌入 + 相似度搜索 | 按需检索，保留关键信息 | 需要向量存储，检索延迟 | 大规模知识库 |
| **滑动窗口** | 保留最近 N 条消息 | 简单，零成本 | 丢失早期上下文 | 短对话场景 |
| **关键信息提取** | 规则/LLM 提取实体、决策、任务 | 高价值信息保留 | 需要定义提取规则 | 任务型对话 |
| **分层记忆** | 短期/长期记忆分离 | 结构清晰，灵活 | 实现复杂 | 多 Agent 协作 |

### 2.2 现有方案对比

#### LangChain Memory
```python
# 内置压缩器
from langchain.memory import ConversationSummaryMemory
from langchain.memory.compressor import LLMChainExtractor

memory = ConversationSummaryMemory(
    llm=llm,
    memory_key="chat_history",
    return_messages=True
)
```
- **优点：** 成熟生态，多种压缩器可选，文档完善
- **缺点：** 重量级，Python 依赖，学习曲线
- **适用性：** ⭐⭐⭐⭐ 适合复杂场景

#### LlamaIndex
```python
from llama_index.core import VectorStoreIndex, SimpleDirectoryReader
from llama_index.core.memory import ChatMemoryBuffer

index = VectorStoreIndex.from_documents(documents)
```
- **优点：** 强大的 RAG 能力，查询引擎灵活
- **缺点：** 学习曲线陡峭，配置复杂
- **适用性：** ⭐⭐⭐ 适合知识库场景

#### Mem0 (新兴方案)
```python
from mem0 import Memory

memory = Memory()
memory.add("用户喜欢 Python", user_id="user_123")
relevant = memory.search("用户偏好", user_id="user_123")
```
- **优点：** 专为 Agent 记忆设计，API 简洁，支持用户级隔离
- **缺点：** 较新，社区小，文档不完善
- **适用性：** ⭐⭐⭐⭐⭐ 高度匹配需求

#### AutoGen Memory
- **优点：** 多 Agent 原生支持，微软背书
- **缺点：** 绑定 AutoGen 框架，灵活性低
- **适用性：** ⭐⭐ 如采用 AutoGen 则适合

### 2.3 开源 vs 自研

| 维度 | 开源方案 | 自研方案 |
|------|----------|----------|
| **开发时间** | 1-2 天集成 | 5-10 天开发 |
| **可控性** | 中（受框架限制） | 高（完全定制） |
| **维护成本** | 低（社区维护） | 高（自行维护） |
| **灵活性** | 中 | 高 |
| **推荐** | ✅ 优先使用 | 特殊需求时自研 |

**建议：** 基于 Mem0 或 LangChain 进行二次开发，平衡效率与灵活性。

## 三、内部需求分析

### 3.1 ai-council 框架集成点

根据现有架构，压缩模块可集成在以下位置：

```
┌─────────────────────────────────────────────────────┐
│                  ai-council 调度层                    │
├─────────────────────────────────────────────────────┤
│  CEO → Nia → PM → Agent Teams                       │
│                      ↓                              │
│         ┌──────────────────────┐                    │
│         │  Context Manager     │ ← 新增模块          │
│         │  ├─ Compression      │                    │
│         │  ├─ Vector Store     │                    │
│         │  └─ Retrieval        │                    │
│         └──────────────────────┘                    │
│                      ↓                              │
│         ┌──────────────────────┐                    │
│         │   Agent Sessions     │                    │
│         │   (ChatGPT/Gemini/   │                    │
│         │    Qwen/DeepSeek...) │                    │
│         └──────────────────────┘                    │
└─────────────────────────────────────────────────────┘
```

**集成点：**
1. **会话管理层** - 每个 Agent 会话独立压缩
2. **跨 Agent 共享记忆** - 项目级上下文共享
3. **MEMORY.md 同步** - 压缩结果归档到长期记忆

### 3.2 多 Agent 系统上下文管理

**场景分析：**

| 场景 | 上下文类型 | 压缩策略 |
|------|------------|----------|
| 单 Agent 长对话 | 连续对话历史 | 摘要 + 滑动窗口 |
| 多 Agent 协作 | 跨 Agent 消息 | 关键信息提取 + 向量检索 |
| 项目级上下文 | 任务、决策、文档 | 分层记忆（项目→任务→对话） |
| 用户偏好记忆 | 长期偏好、习惯 | 结构化存储（JSON） |

### 3.3 MEMORY.md 智能压缩

**当前状态：** 手动维护，无自动化

**改进方案：**
```
原始对话 → 压缩器 → 结构化记忆项 → MEMORY.md 追加
                    ↓
            {
              "type": "decision|task|preference|lesson",
              "content": "...",
              "timestamp": "...",
              "agents": ["nia", "codex"],
              "project": "AI 代码可视化"
            }
```

**自动化规则：**
- 检测到决策关键词 → 提取到"决策"分类
- 检测到任务分配 → 提取到"进行中项目"
- 检测到用户偏好 → 提取到"用户偏好"
- 每日会话结束 → 生成摘要追加到 `memory/YYYY-MM-DD.md`

### 4.2 核心模块设计

#### Module 1: ContextCompressor
```typescript
interface CompressionConfig {
  strategy: 'summary' | 'vector' | 'hybrid';
  maxTokens: number;
  retentionWindow: number; // 保留最近 N 条
  extractEntities: boolean;
}

class ContextCompressor {
  compress(messages: Message[], config: CompressionConfig): CompressedContext;
  extractKeyInfo(messages: Message[]): KeyInformation;
  generateSummary(messages: Message[]): string;
}
```

#### Module 2: VectorMemory
```typescript
class VectorMemory {
  addEmbedding(text: string, metadata: Metadata): string;
  search(query: string, limit: number): SearchResult[];
  deleteOlderThan(days: number): void;
}
```

#### Module 3: MemorySync
```typescript
class MemorySync {
```

### 5.2 合并开发建议

**不建议合并**，原因：
1. **产品定位不同**：代码可视化是前端产品，上下文压缩是后端基础设施
2. **技术栈差异**：可视化需要 Canvas/SVG，压缩需要向量/存储
3. **部署方式不同**：可视化是 Vercel 静态部署，压缩需要服务端运行

**建议：**
- 作为独立模块开发：`@oneai/context-compression`
- 共享工具库：API 客户端、配置管理、日志工具
- 统一项目结构：遵循公司代码规范

### 5.3 潜在协同点

1. **可视化调试界面**（可选）
   - 上下文压缩效果可视化
   - 向量检索结果展示
   - 记忆图谱浏览

2. **共享配置中心**
   - 统一 API 密钥管理
