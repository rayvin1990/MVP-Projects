## 📋 咨询发起

### 发起方式

| 发起人 | 方式 | 说明 |
|--------|------|------|
| **主任** | 直接告诉 Nia | 最高优先级 |
| **Nia** | 心跳检查发现 | 需主任确认 |

### 议题分类

| 类型 | 响应时间 | 示例 |
|------|----------|------|
| **紧急** | 2 小时内 | 高风险决策、突发问题 |
| **常规** | 24 小时内 | 产品立项、市场策略 |
| **复杂** | 48 小时内 | 战略方向、组织调整 |

## 🔄 咨询流程

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 议题整理                                        │
│  - Nia 与主任确认议题核心                               │
│  - 明确决策类型和期望输出                               │
│  - 确定参与模型（默认全体）                             │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: 记忆提取                                        │
│  - 从共享记忆区提取相关背景                             │
│  - 公司背景（200 字内）                                  │
│  - 相关决策历史                                         │
│  - 相关项目进展                                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: 并行咨询                                        │
│  - 向 6 模型发送相同议题 + 记忆注入                      │
│  - 各模型独立分析（避免互相影响）                       │
│  - 设定回复截止时间                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 4: 汇总整合                                        │
│  - Nia 提炼共识点                                       │
│  - Nia 提炼分歧点                                       │
│  - Nia 记录独特视角                                     │
│  - Nia 计算综合得分                                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 5: 提交主任                                        │
│  - 输出结构化建议报告                                   │
│  - 主任审阅并决策                                       │
│  - 批准/否决/要求补充                                   │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 6: 归档记忆                                        │
│  - 创建决策归档文档                                     │
│  - 更新相关项目进展                                     │
│  - 记录模型表现                                         │
└─────────────────────────────────────────────────────────┘
```

```
{注入公司背景，200 字内}

{注入相关决策历史}

{注入相关项目进展}

【本次议题】
{详细描述议题和需要分析的问题}

【期望输出】
1. 你的核心建议（1-2 句话）
2. 详细分析（3-5 点）
3. 风险评估（如有）
4. 推荐行动（1-3 步）

【截止时间】
{YYYY-MM-DD HH:MM}
```

## 📁 输出文档

### 必须创建

- `共享记忆/决策历史/YYYY-MM-DD-议题名称.md`

### 必须更新

- `共享记忆/项目进展/相关项目.md`（如有进展更新）

### 可选创建

- `共享记忆/模型表现档案/模型名称.md`（月度评估时）

---

## ⚠️ 特殊情况处理

### 模型未响应

- 等待超过最迟时间 → 标记为"未响应"
- 继续汇总其他模型建议
- 记录到模型表现档案

### 模型回复质量低

- 记录到模型表现档案
- 可要求模型重新分析（消耗额外额度）
- 或 Nia 补充分析

### 主任要求补充

- Nia 整理补充问题
- 重新咨询相关模型
- 更新建议报告
