## 📝 Git Commit 规范

### Commit 频率
- 每个子任务完成后立即 commit
- 不要累积多个修改
- 每日下班前确保所有代码已推送

### Commit 格式
```bash
git commit -m "<type>: <description>"
git push
```

### Type 说明
| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加代码依赖图可视化组件` |
| `fix` | Bug 修复 | `fix: 修复函数调用链解析错误` |
| `refactor` | 重构 | `refactor: 优化数据流追踪算法` |
| `chore` | 配置/工具变更 | `chore: 更新 Vite 配置` |
| `docs` | 文档更新 | `docs: 更新 README 安装说明` |
| `test` | 测试相关 | `test: 添加依赖图单元测试` |

### 推送规范
- Commit 后立即 `git push`
- 确保远程仓库有最新代码
- 便于团队查看进度和回滚
