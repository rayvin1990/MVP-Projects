## 📋 工作流程

### 测试任务

```
接收任务 → 编写测试用例 → 执行测试 → 报告结果 → Bug 跟踪
```

### 安全任务

```
接收任务 → 漏洞扫描 → 风险评级 → 修复建议 → 复测验证
```

### 数据分析任务

```
接收任务 → 数据收集 → 清洗处理 → 分析洞察 → 可视化报告
```

## 📝 Git Commit 规范

### Commit 频率
- 每个子任务完成后立即 commit
- 不要累积多个修改
- 每日下班前确保所有代码已推送

### Commit 格式
```bash
git commit -m "<type>: <description>"
git push
```

### Type 说明
| Type | 说明 | 示例 |
|------|------|------|
| `feat` | 新功能 | `feat: 添加安全扫描脚本` |
| `fix` | Bug 修复 | `fix: 修复测试用例断言错误` |
| `refactor` | 重构 | `refactor: 优化数据分析流程` |
| `chore` | 配置/工具变更 | `chore: 更新测试依赖` |
| `docs` | 文档更新 | `docs: 更新测试报告模板` |
| `test` | 测试相关 | `test: 添加边界条件测试` |

### 推送规范
- Commit 后立即 `git push`
- 确保远程仓库有最新代码
- 便于团队查看进度和回滚

### 测试/安全任务特殊说明
- 测试用例新增/修改 → `test:`
- 安全脚本更新 → `feat:` 或 `fix:`
- 扫描报告更新 → `docs:`
