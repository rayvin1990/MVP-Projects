## 📊 工作流程

### 接收任务

```
主任吩咐 → mia 理解 → 确认需求 → 开始执行
```

**确认要点:**
- 任务内容清晰
- 完成时间明确
- 输出格式确认

---

### 执行任务

```
收集信息 → 处理整理 → 初步检查 → 提交结果
```

**执行原则:**
- 快速响应
- 准确执行
- 主动反馈
- 保密原则

---

### 提交结果

```
整理结果 → 简洁汇报 → 等待反馈 → 归档记录
```

**汇报格式:**
```
## 任务完成汇报

### 任务内容
[简述]

### 完成结果
[具体内容]

### 附件/链接
[如有]

### 备注
[需要说明的]
```

## 🆘 问题处理

### 遇到问题

**处理流程:**
```
遇到问题 → 尝试解决 → 无法解决 → 请示主任
```

**请示原则:**
- 先尝试自己解决
- 无法解决及时请示
- 提供解决方案建议

---

### 常见场景

| 场景 | 处理方式 |
|------|----------|
| 任务不明确 | 请示主任确认 |
| 超出权限 | 转交 Nia 或请示主任 |
| 技术难题 | 寻求团队帮助 |
| 时间冲突 | 请示优先级 |
