# 交付标准（Definition of Done）

**版本：** v1.0  
**生效日期：** 2026-03-12  
**适用范围：** 所有 AI 团队成员

---

## ✅ 通用 DoD（所有任务）

任何任务交付前必须满足：

### 1. 功能完成
- [ ] 任务要求的功能全部实现
- [ ] 自测通过（开发者验证）

### 2. 质量保证
- [ ] 代码审查通过（CC 负责）
- [ ] 无已知 P0/P1 Bug
- [ ] 测试覆盖率 ≥80%（核心功能）

### 3. 文档完整
- [ ] README/CHANGELOG 已更新
- [ ] 使用说明清晰
- [ ] 已知问题已记录

### 4. 用户视角检查
- [ ] 从用户角度验证流程顺畅
- [ ] 文案清晰，无开发者术语
- [ ] 错误提示友好、可操作

### 5. 知识沉淀
- [ ] 经验教训存入共享记忆区
- [ ] 可复用的代码/模板已归档

---

## 📋 角色特定 DoD

### 市场研究员

- [ ] 调研样本≥5 个目标用户（或等效数据）
- [ ] 竞品分析≥3 个
- [ ] 所有结论有数据支撑
- [ ] 明确"做/不做"建议
- [ ] 报告存入共享记忆区

### Google AI Studio（UI/UX）

- [ ] 设计前参考设计系统
- [ ] 提供≥3 个方案供选择
- [ ] 使用 CSS 变量（便于主题切换）
- [ ] 响应式设计（移动端适配）
- [ ] 无障碍检查（基本 a11y）

### CC（研发经理/开发）

- [ ] 先确认技术方案再编码
- [ ] 遵循代码规范（lint 通过）
- [ ] 单元测试≥80%
- [ ] 无安全漏洞（扫描通过）
- [ ] 性能测试通过（加载<3 秒）

### Codex（测试/安全/数据）

- [ ] 测试用例覆盖核心流程
- [ ] 安全扫描无高危漏洞
- [ ] 数据分析有清洗步骤记录
- [ ] 报告有风险等级标注

### DP（文档经理）

- [ ] 文档结构清晰
- [ ] 示例完整可运行
- [ ] 截图/图表清晰
- [ ] 无拼写/语法错误

---

## 🚫 不满足 DoD 的处理

**任何不满足 DoD 的交付：**

1. **退回重做** - 不验收，不计入完成
2. **记录原因** - 存入共享记忆区（避免重复）
3. **主任可见** - 透明化质量问题
