# 验收标准

**版本：** v1.0  
**生效日期：** 2026-03-12  
**适用范围：** 所有交付物

---

## 🔄 验收流程

```
提交 → 初审（Nia） → 终审（主任） → 归档
```

---

## ✅ 自动化验收（Codex 负责）

### 代码类交付物

| 检查项 | 标准 | 工具 |
|--------|------|------|
| 代码风格 | ESLint/Prettier 通过 | eslint, prettier |
| 测试覆盖率 | ≥80% | jest, vitest |
| 安全漏洞 | 无高危 | npm audit, security-scan.ps1 |
| 构建成功 | 无错误 | npm run build |
| 类型检查 | TypeScript 通过 | tsc |

### 文档类交付物

| 检查项 | 标准 | 工具 |
|--------|------|------|
| 格式正确 | Markdown 语法通过 | markdownlint |
| 结构完整 | 必需章节存在 | 自定义脚本 |
| 无错别字 | 拼写检查通过 | cspell |
| 链接有效 | 无 404 | 自定义脚本 |

### UI 类交付物

| 检查项 | 标准 | 工具 |
|--------|------|------|
| 设计一致性 | 符合设计系统 | 人工 + 截图比对 |
| 响应式 | 移动端正常 | 人工测试 |
| 无障碍 | 基本 a11y 通过 | axe-core |

---

## 👤 人工验收

### 初审（Nia）

**检查内容：**
- [ ] DoD 检查清单完整
- [ ] 自动化验收通过
- [ ] 交付物格式正确
- [ ] 文档完整

**结果：**
- ✅ 通过 → 提交主任终审
- ❌ 不通过 → 退回重做

---

### 终审（主任）

**检查内容：**
- [ ] 功能符合预期
- [ ] 用户体验顺畅
- [ ] 达到"尖叫"标准
- [ ] 可以发布

**结果：**
- ✅ 通过 → 归档 + 部署
- ❌ 不通过 → 反馈修改意见

## 🎯 验收标准细则

### 设计验收

| 维度 | 标准 |
|------|------|
| 视觉一致性 | 符合设计系统（颜色/字体/间距） |
| 交互合理 | 用户操作符合直觉 |
| 响应式 | 手机/平板/桌面正常 |
| 加载速度 | 首屏<3 秒 |

### 代码验收

| 维度 | 标准 |
|------|------|
| 功能实现 | 需求 100% 实现 |
| 可维护性 | 代码清晰、有注释 |
| 性能 | 无性能瓶颈 |
| 安全 | 无已知漏洞 |

### 市场报告验收

| 维度 | 标准 |
|------|------|
| 方法科学 | 样本量足够、方法合理 |
| 数据支撑 | 结论有数据，非 AI 幻觉 |
| 尖叫点 | 明确用户愿意付费的点 |
| 可执行 | 建议具体、可落地 |

---

## 🚫 不通过的处理

**任何不通过的交付：**

1. **记录问题** - 验收报告详细记录
2. **退回重做** - 负责人修改后重新提交
3. **存入共享记忆区** - 避免重复错误
4. **主任可见** - 透明化质量
